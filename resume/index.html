Open Source

MooTools (2006-present)

I am the lead developer & founder of the MooTools JavaScript framework. MooTools is being actively developed by a small team of very smart people. I manage all code-related contributions, patches, and enhancements. I am also the developer and designer of the MooTools website. The source code for MooTools is available on github.

QuickStart (2014-present)

During my stay at Spotify I have developed QuickStart, a commonJS compiler for node.js and browsers. While similar in scope to Browserify, QuickStart does not require any pre-compilation during development, and can be used to seamlessly make use of commonJS modules in the browsers, making development quick and easy. QuickStart is still used at Spotify to develop and publish applications on the Spotify Desktop Client. The source code for QuickStart is available on github.

MooFx (2005-2013)

MooFx is a javascript animation framework built to support and enhance css3 transitions. I created the original version of MooFx in 2005. In 2012 MooFx was revamped to support css3 transitions. Though MooFx is mostly an animation framework, it is also used to set and retrieve a normalized version of CSS styles from DOM nodes. MooFx is fully able to animate transforms, even in browsers without support for css3 transitions, as it implements the W3C specification for Transitions and animations between transform values in plain client-side code. The source code for MooFx is available on github.

ART (2007-2012)

ART is a framework for working with, and creating vector images. ART natively supports SVG and VML, providing a standardized API. ART prefers to use SVG where available, but falls back to VML in Internet Explorer < 9. The source code for ART is available on github.

WrapUp (2012-present)

WrapUp is a CommonJS 1.0 modules compiler and dependency mapper. WrapUp is used mainly to create a single JavaScript file out of a list of required modules. WrapUp is written in JavaScript, on the node.js platform. The source code for WrapUp is avaliable on github.

Slick (2007-present)

Slick is a standalone DOM selector engine for any web browser. I wrote Slick in collaboration with MooTools developers to have a fast, solid way to retrieve DOM nodes using CSS syntax. The source code for Slick is available on github.

Packager (2007-2010)

Packager is a dependency resolver program built with PHP5 that I wrote in 2007 to support the MooTools online javascript builder. Packager is now being phased out and replaced with WrapUp. The source code for packager is available on github.
